# 微信支付集成设计文档

## 1. 概述

在现有订单创建流程后接入微信支付（商户平台 JSAPI 支付），用户提交订单后调起微信支付，支付完成后更新订单状态。

### 核心流程

```
提交订单 → 创建订单（待支付）
  → 请求支付参数（后端统一下单）
    → 调起 wx.requestPayment
      → 支付成功 → 更新订单状态 + 跳转结果页
      → 支付失败/取消 → 留在订单详情页
```

## 2. 微信支付流程

### 2.1 时序图

```
小程序                    后端                     微信支付平台
  │                       │                         │
  │ POST /v1/order/create │                         │
  │───────────────→───────│                         │
  │←──── { orderId } ────│                         │
  │                       │                         │
  │ POST /v1/order/pay   │                         │
  │───────────────→───────│                         │
  │                       │ POST unifiedorder      │
  │                       │──────────────→──────────│
  │                       │← { prepay_id } ────────│
  │←── { paySign, ... } ─│                         │
  │                       │                         │
  │ wx.requestPayment     │                         │
  │───────────────→───────│(微信客户端直接调起支付)   │
  │←── 支付结果回调 ────│                         │
  │                       │                         │
  │ POST /v1/order/confirm (支付结果通知到后端)       │
  │              或 微信异步通知                      │
  │                       │                         │
```

### 2.2 微信小程序端

- 使用 `wx.requestPayment` 调起微信支付
- 需要从后端获取 **支付参数**（`timeStamp`, `nonceStr`, `package`, `signType`, `paySign`）
- 支付结果通过 `wx.requestPayment` 的 `success`/`fail` 回调获取
- **注意**：`success` 回调不代表支付一定成功，需要以后端异步通知或主动查询为准

## 3. API 设计

### 3.1 POST /v1/order/pay — 获取支付参数

**请求体：**

```json
{
  "orderId": "CH2026061400000001",
  "paymentMethod": "wechat"
}
```

**逻辑：**

```
1. 校验订单是否存在、是否属于当前用户、支付状态是否为 0（未支付）
2. 调用微信统一下单 API（unifiedorder），获取 prepay_id
3. 根据 prepay_id 生成前端需要的支付签名参数
4. 返回支付参数
```

**后端响应：**

```json
{
  "flag": true,
  "resData": {
    "orderId": "CH2026061400000001",
    "payAmount": 100,
    "paySign": {
      "timeStamp": "1746800000",
      "nonceStr": "abc123",
      "package": "prepay_id=wx...",
      "signType": "MD5",
      "paySign": "signature..."
    }
  }
}
```

### 3.2 POST /v1/order/pay/notify — 微信支付回调（异步通知）

- 微信支付平台在用户支付成功后会异步通知到该地址
- 请求体为微信支付的 XML 格式
- 收到通知后验证签名，更新订单状态为已支付

### 3.3 GET /v1/order/check-pay — 主动查询支付状态

**请求参数：** `orderId`

**响应：**

```json
{
  "flag": true,
  "resData": {
    "orderId": "CH2026061400000001",
    "payStatus": 1,
    "payAmount": 100,
    "paidAt": "2026-06-14 23:55:00"
  }
}
```

## 5. 小程序端支付流程

### 5.1 支付调用

```javascript
// pages/order/order-confirm/pay.js

import { post } from '../../../utils/request';

function getPayParams(orderId) {
  return post('/v1/order/pay', { orderId });
}

export const wechatPay = (orderId) => {
  return getPayParams(orderId).then(function(res) {
    var paySign = res.paySign;
    if (!paySign) return Promise.reject(new Error('获取支付参数失败'));
    return new Promise(function(resolve, reject) {
      wx.requestPayment({
        timeStamp: paySign.timeStamp,
        nonceStr:  paySign.nonceStr,
        package:   paySign.package,
        signType:  paySign.signType,
        paySign:   paySign.paySign,
        success:   function() { resolve(res); },
        fail:      function(err) { reject(err); },
        complete:  function() {},
      });
    });
  });
};
```

### 5.2 对接现有流程

修改 `pages/order/order-confirm/pay.js` 中的 `wechatPayOrder`：

### 5.3 支付结果处理

```javascript
// 支付成功的回调
wx.requestPayment({
  success: function(payRes) {
    // payRes = { errMsg: "requestPayment:ok" }
    // 支付成功 → 查询订单确认支付状态 → 跳转结果页
    checkPayStatus(orderId).then(function(orderInfo) {
      paySuccess({ tradeNo: orderId, payAmt: orderInfo.payAmount });
    });
  },
  fail: function(err) {
    // err.errMsg 可能为 "requestPayment:fail cancel"（用户取消）
    payFail(payOrderInfo, err.errMsg);
  },
});
```

### 6.2 统一下单接口调用

```python
import requests
import hashlib
import time
import random
import string
import xml.etree.ElementTree as ET

def _wechat_unified_order(out_trade_no, total_fee, openid, notify_url):
    """调用微信支付统一下单"""
    params = {
        'appid':        _get_config('wechat_app_id'),
        'mch_id':       _get_config('wechat_mch_id'),
        'nonce_str':    ''.join(random.choices(string.ascii_letters + string.digits, k=32)),
        'body':         '商城-商品',
        'out_trade_no': out_trade_no,
        'total_fee':    str(total_fee),  # 单位：分
        'spbill_create_ip': '127.0.0.1',
        'notify_url':   notify_url,
        'trade_type':   'JSAPI',
        'openid':       openid,
    }
    # 生成签名
    sign = _generate_sign(params, _get_config('wechat_mch_key'))
    params['sign'] = sign
    # 转为 XML 发送
    xml_data = _dict_to_xml(params)
    resp = requests.post('https://api.mch.weixin.qq.com/pay/unifiedorder',
                         data=xml_data, headers={'Content-Type': 'text/xml'})
    result = _xml_to_dict(resp.text)
    if result.get('return_code') != 'SUCCESS' or result.get('result_code') != 'SUCCESS':
        raise Exception('微信统一下单失败: ' + result.get('return_msg', ''))
    return result['prepay_id']
```

### 6.3 生成前端支付签名

```python
def _build_pay_sign(prepay_id):
    """生成前端 wx.requestPayment 需要的签名"""
    params = {
        'appId':     _get_config('wechat_app_id'),
        'timeStamp': str(int(time.time())),
        'nonceStr':  ''.join(random.choices(string.ascii_letters + string.digits, k=32)),
        'package':   'prepay_id=' + prepay_id,
        'signType':  'MD5',
    }
    params['paySign'] = _generate_sign(params, _get_config('wechat_mch_key'))
    return params
```

### 6.4 签名算法

```python
def _generate_sign(params, key):
    """微信支付 MD5 签名"""
    keys = sorted(params.keys())
    raw = '&'.join(['{}={}'.format(k, params[k]) for k in keys if params[k] != ''])
    raw += '&key=' + key
    return hashlib.md5(raw.encode('utf-8')).hexdigest().upper()
```
