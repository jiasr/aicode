# 配送方式与运费管理设计文档

## 1. 概述

配送方式由管理后台统一配置，用户下单时根据商品的配送方式自动匹配，无需手动选择。系统根据配送方式和收货地址计算运费，生成含运费的订单。

### 核心流程

```
管理后台配置配送方式
  ├── 系统设置 → 设置商店支持的配送方式（快递/同城配送/自提）
  ├── 运费模板管理 → 创建快递运费模板
  └── 商品编辑 → 选择运费模板 → 确定配送方式
        ↓
商品详情页展示配送方式
订单确认页 → 系统自动匹配配送方式
  ├── 快递 → 用户选地址 → 运费模板自动算运费 → 下单支付
  ├── 同城配送 → 用户选地址+时段 → 按区域计费 → 下单支付
  └── 自提 → 用户选门店 → 运费0 → 下单支付
```

## 2. 配送方式设计

### 2.1 配送方式类型

| delivery_type | 名称 | 计费逻辑 | 用户操作 |
|:---:|---|---|---|
| 0 | **快递** | 按运费模板计算（见第3章） | 选择收货地址 |
| 1 | **同城配送** | 按区域/距离固定费率（后续迭代） | 选择地址 + 配送时段 |
| 2 | **自提** | 免费 | 选择门店 |

### 2.2 配置方式

配送方式按商品级别配置，每个商品关联一个**运费模板**，模板决定了该商品的配送方式：

| 场景 | 配置方式 |
|---|---|
| **仅快递** | 商品关联快递运费模板 → `delivery_type=0` |
| **仅自提** | 商品关联自提模板（特殊模板） → `delivery_type=2` |
| **快递+自提** | 商品同时关联快递模板和自提标记 → 下单时让用户选门店或地址 |
| **仅同城** | 商品关联同城配送模板 → `delivery_type=1` |

> 初始版本先支持"仅快递"模式，商品默认 `delivery_type=0`，后续再开放自提和同城配送。

## 3. 快递运费管理

本章仅适用于 `delivery_type=0`（快递）的场景。

### 3.1 计费模式

| pricing_type | 模式 | 说明 |
|:---:|---|---|
| 0 | **固定运费** | 每单统一收取固定金额，不随件数变化 |
| 1 | **按件收费** | 首件 X 元 + 续件 Y 元/件 |

> 按重计费暂不支持，后续版本可扩展 `pricing_type=2`。

### 3.2 条件包邮

支持两种包邮条件，优先级：指定地区包邮 > 满额包邮

| 条件 | 说明 |
|---|---|
| **指定地区包邮** | 某些省份始终免运费（如西藏） |
| **满额包邮** | 商品金额 ≥ 门槛值则免运费 |

### 3.3 地区差异化

每个运费模板下可对各省份配置差异化定价：
- 该地区包邮
- 覆盖固定运费/首费/续费
- 覆盖包邮门槛

未配置的省份使用模板默认值。

### 3.4 数据模型

#### 运费模板表 `t_mall_freight_template`

```sql
CREATE TABLE `t_mall_freight_template` (
  `id`              INT PRIMARY KEY AUTO_INCREMENT,
  `name`            VARCHAR(100) NOT NULL               COMMENT '模板名称，如"江浙沪8元"',
  `pricing_type`    INT DEFAULT 1                       COMMENT '计费方式 0=固定运费 1=按件',
  `fixed_fee`       INT DEFAULT 0                       COMMENT '固定运费金额(分)，pricing_type=0时生效',
  `first_unit`      INT DEFAULT 1                       COMMENT '首件数量，pricing_type=1时生效',
  `first_fee`       INT DEFAULT 0                       COMMENT '首件费用(分)',
  `continue_unit`   INT DEFAULT 1                       COMMENT '续件数量',
  `continue_fee`    INT DEFAULT 0                       COMMENT '续件费用(分)',
  `free_threshold`  INT DEFAULT 0                       COMMENT '满额包邮门槛(分)，0=不支持满额包邮',
  `is_default`      INT DEFAULT 0                       COMMENT '是否默认模板 1=是 0=否',
  `create_time`     DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time`     DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='运费模板';
```

#### 运费地区规则表 `t_mall_freight_region`

```sql
CREATE TABLE `t_mall_freight_region` (
  `id`              INT PRIMARY KEY AUTO_INCREMENT,
  `template_id`     INT NOT NULL                        COMMENT '关联模板ID',
  `region_code`     VARCHAR(20) NOT NULL                COMMENT '省份编码(与regions表中code对应)',
  `region_name`     VARCHAR(50) NOT NULL                COMMENT '省份名称',
  `is_free`         INT DEFAULT 0                       COMMENT '是否包邮 1=是 0=按规则',
  `fixed_fee`       INT DEFAULT NULL                    COMMENT '该地区固定运费(分)，覆盖模板值',
  `first_fee`       INT DEFAULT NULL                    COMMENT '该地区首费(分)，覆盖模板值',
  `continue_fee`    INT DEFAULT NULL                    COMMENT '该地区续费(分)，覆盖模板值',
  `free_threshold`  INT DEFAULT NULL                    COMMENT '该地区包邮门槛(分)，覆盖模板值',
  INDEX `idx_region_template` (`template_id`),
  INDEX `idx_region_code` (`region_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='运费地区规则';
```

### 3.5 运费计算逻辑

```
输入: delivery_type, items, provinceCode, totalAmount
输出: freightAmount

if delivery_type == 2 (自提):
    return 0

if delivery_type == 1 (同城配送):
    return 同城配送费(暂固定)  // 后续可扩展

if delivery_type == 0 (快递):
    return calcExpressFreight(items, provinceCode, totalAmount)
```

**`calcExpressFreight` 详细逻辑：**

```
Step 1 — 获取运费模板
  对 items 中的每个商品，取 spu.freight_template_id
  若均为 0 → 取系统默认模板(is_default=1)
  若存在多种模板 → 取运费最高的

Step 2 — 获取地区规则
  region = FreightRegion(template_id, provinceCode)

Step 3 — 判断包邮
  if region and region.is_free == 1: return 0
  threshold = region.free_threshold ?? template.free_threshold
  if threshold > 0 and totalAmount >= threshold: return 0

Step 4 — 计算运费
  case template.pricing_type:
    0: return region.fixed_fee ?? template.fixed_fee
    1:
      first_fee    = region.first_fee    ?? template.first_fee
      continue_fee = region.continue_fee ?? template.continue_fee
      totalQty     = sum(items.quantity)
      remain       = max(0, totalQty - template.first_unit)
      return first_fee + ceil(remain / template.continue_unit) * continue_fee
```

### 3.6 计算示例

**例1：固定运费模式**

| 配置 | 值 |
|------|-----|
| 模板 | 固定运费 20元 |
| 新疆地区规则 | 固定运费 30元 |
| 西藏 | 包邮 |

```
买3件 → 送到广东 → 20元   （无地区规则，用模板默认）
买3件 → 送到新疆 → 30元   （新疆地区规则覆盖）
买3件 → 送到西藏 → 0元    （包邮）
```

**例2：按件计费模式**

| 配置 | 值 |
|------|-----|
| 模板 | 首件10元 续件5元/件 满99元包邮 |
| 北京地区规则 | 首件8元 续件3元/件 |
| 广东省 | 满50元包邮 |

```
1件 → 送到上海 → 10元       （首件，未满99）
3件 → 送到上海 → 10+5×2=20元
3件 → 送到北京 → 8+3×2=14元 （北京地区规则）
3件总价120 → 送到上海 → 0元 （满99包邮）
3件总价30 → 送到广东 → 10+5×2=20元（未满50包邮门槛）
3件总价60 → 送到广东 → 0元  （满50包邮）
```

## 7. 自提核销流程

```
用户下单（自提） → 生成6位核销码 → 到店出示核销码
  → 管理员在后台输入核销码 → 核销成功 → 订单状态变为"已完成"
```

核销码规则：6位数字/字母随机，与订单号关联，设置有效期（默认48小时）。

### 核销 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/v1/order/admin/pickup/verify` | 管理员核销：输入核销码，验证有效后更新订单状态 |

## 8. 实施步骤

| Phase | 内容 | 涉及 |
|:---:|------|------|
| 0 | **基础框架**：Order 表扩字段 + GoodsSpu 扩字段 + 系统配置 | 后端 |
| 1 | **快递运费模板**：模板/地区表 + 计算引擎 + 管理后台 API + 管理页面 | 全端 |
| 2 | **订单预览接入运费**：`preview()`/`create()` 计算并写入运费 | 后端 + 小程序 |
| 3 | **自提门店**：门店表 + 自提流程 + 核销（后续迭代） | 全端 |
| 4 | **同城配送**：时段选择 + 计费规则（后续迭代） | 全端 |

> 当前版本聚焦 Phase 0~2：配送方式框架 + 快递运费模板 + 订单接入运费。Phase 3~4 后续迭代。
