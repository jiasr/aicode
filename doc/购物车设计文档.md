# 购物车功能设计文档

## 1. 概述

服务端持久化购物车：用户登录后购物车数据存于服务端，支持多端同步。

### 核心流程

```
[商品列表/详情] → 选SKU → 加入购物车 → 购物车页 → 选中结算 → 下单
```

---

## 2. 数据模型

### 2.1 表结构

```sql
CREATE TABLE t_mall_cart (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    user_id     VARCHAR(255) NOT NULL              COMMENT '用户ID',
    spu_id      VARCHAR(255) NOT NULL              COMMENT '商品SPU ID',
    sku_id      VARCHAR(255) NOT NULL              COMMENT '商品SKU ID',
    quantity    INT NOT NULL DEFAULT 1             COMMENT '加购数量',
    is_selected TINYINT(1) DEFAULT 1               COMMENT '是否选中 1是 0否',
    create_time DATETIME DEFAULT NOW()             COMMENT '创建时间',
    update_time DATETIME DEFAULT NOW() ON UPDATE NOW() COMMENT '更新时间',
    UNIQUE KEY uk_user_sku (user_id, sku_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='购物车表';
```

### 2.2 关联查询

购物车列表查询时联表查 SPU/SKU 信息：

```
t_mall_cart
  ├─ JOIN t_mall_goods_sku ON cart.sku_id = sku.id  → 获取 SKU价格/库存/规格
  └─ JOIN t_mall_goods_spu ON cart.spu_id = spu.id  → 获取 SPU标题/主图
```

## 3. API 设计

| 方法 | 路径 | 描述 |
|------|------|------|
| GET  | `/v1/cart/list` | 获取用户购物车列表 |
| POST | `/v1/cart/add` | 添加商品到购物车 |
| POST | `/v1/cart/update` | 更新数量/选中状态 |
| POST | `/v1/cart/delete` | 删除购物车项 |
| POST | `/v1/cart/clear` | 清空购物车 |

### 3.1 GET /v1/cart/list

请求头带 `token`（识别用户）。

**响应：**

```json
{
  "flag": true,
  "resData": {
    "items": [
      {
        "cartId": 1,
        "spuId": "xxx",
        "skuId": "yyy",
        "title": "新疆哈密瓜",
        "thumb": "http://...",
        "specLabel": "5斤",
        "price": 2990,
        "stock": 100,
        "quantity": 2,
        "isSelected": true
      }
    ],
    "totalPrice": 5980,
    "totalCount": 2
  }
}
```

### 3.2 POST /v1/cart/add

**请求体：**

```json
{
  "spuId": "xxx",
  "skuId": "yyy",
  "quantity": 1
}
```

**逻辑：**
- 查库存：quantity > stock → 返回错误
- 该 SKU 已在购物车 → quantity += 请求数量
- 不在购物车 → 新增记录
- 返回最新购物车数量

### 3.3 POST /v1/cart/update

```json
{
  "cartId": 1,
  "quantity": 3,        // 可选
  "isSelected": true     // 可选
}
```

### 3.4 POST /v1/cart/delete

```json
{
  "cartId": 1
}
// 或批量
{
  "cartIds": [1, 2, 3]
}
```

### 3.5 POST /v1/cart/clear

清空当前用户所有购物车项（结算后调用）。

## 4. 前端交互

### 4.1 分类页 — 单规格直接加购

```
用户点击卡片「+」按钮
  → POST /v1/cart/add { spuId, skuId: skuList[0].skuId, quantity: 1 }
  → success: Toast "已加入购物车" + 更新 tabBar badge
  → fail: 提示库存不足
```

### 4.2 分类页 — 多规格弹窗选后加购

```
用户点击「选规格」
  → 弹出规格选择面板
  → 选完规格 → 点击「加入购物车」
  → POST /v1/cart/add
```

### 4.3 详情页 — 加购

```
已存在的规格选择弹窗
  → 选完规格 → 点击「加入购物车」
  → POST /v1/cart/add
```

### 4.4 购物车页

```
进入购物车
  → GET /v1/cart/list → 渲染列表
  → 分组展示（门店？无门店则扁平展示）
  → 支持：
     - 修改数量 (±, 校验库存)
     - 单选/全选
     - 删除单项
     - 清空失效
     - 去结算（携带选中的商品列表）
```

## 5. TabBar Badge 更新

购物车数量存储在 `app.globalData.cartCount`，每次加购成功或进入购物车后更新。

```js
// app.js 全局方法
updateCartBadge() {
  const count = this.globalData.cartCount || 0;
  wx.setTabBarBadge({
    index: 2,  // 购物车 tab 索引
    text: count > 0 ? String(count) : '',
  });
}
```

## 6. 库存校验

| 时机 | 校验规则 |
|------|---------|
| 加购时 | 请求数量 ≤ SKU库存，否则提示"库存不足" |
| 修改数量时 | 新数量 ≤ SKU库存，超限时提示并阻止 |
| 结算时 | 再次校验，库存不足的标记为"失效" |

---

## 7. 商品失效处理

已加入购物车的商品可能因以下原因失效：

| 失效原因 | 检测时机 | 处理方式 |
|---------|---------|---------|
| SKU 库存为 0 | 进入购物车、结算时 | 移入"失效商品"区，不可选中 |
| SPU 已下架 | 进入购物车 | 移入"失效商品"区，仅可删除 |
| SKU 已删除 | 进入购物车 | 自动清除 |
| 加购数量 > 当前库存 | 进入购物车 | 移入"失效商品"区，数量自动修正为最大库存 |

### 7.1 后端实现

```python
def get_cart_with_validation(user_id):
    """获取购物车，同时校验商品有效性"""
    items = query_cart(user_id)
    
    valid_items = []
    invalid_items = []
    
    for item in items:
        spu = get_spu(item.spu_id)
        sku = get_sku(item.sku_id)
        
        # SKU/SPU 不存在 → 自动删除
        if not spu or not sku:
            delete_cart_item(item.id)
            continue
        
        # 下架
        if spu.is_put_on_sale == 0:
            invalid_items.append(build_invalid(item, spu, sku, '商品已下架'))
            continue
        
        # 库存 0
        if sku.stock_quantity <= 0:
            invalid_items.append(build_invalid(item, spu, sku, '商品已售罄'))
            continue
        
        # 数量超库存 → 修正为最大库存，标记失效
        if item.quantity > sku.stock_quantity:
            update_quantity(item.id, sku.stock_quantity)
            invalid_items.append(build_invalid(item, spu, sku, '库存不足'))
            continue
        
        valid_items.append(build_valid(item, spu, sku))
    
    return { 'valid': valid_items, 'invalid': invalid_items }
```

### 7.3 前端交互

```
购物车页展示分为两个区域：

┌─ 正常商品列表 ───────────────┐
│ ○ [图] 商品A  ¥29.9  数量±  │  ← 可选中/改数量/删除
│ ○ [图] 商品B  ¥19.9  数量±  │
├─ 失效商品（可展开） ──────────┤
│ ▽ 失效商品 (2)              │  ← 默认折叠
│   [图] 已下架商品   🔗 找相似 │  ← 只显示，不可操作
│   [图] 已售罄商品    🗑 删除  │
├─ 全选 / 合计 ────────────────┤
│  合计: ¥49.8   去结算(2)     │  ← 只统计正常商品
└──────────────────────────────┘
```

- 失效商品不可选中，不计入结算金额
- 下架商品提供"找相似"入口（跳同类商品列表）
- 售罄商品提供"删除"按钮
- 可一键"清空失效商品"

### 7.4 购物车初始化校验

每次进入购物车页面或从其他页面切回时：

```
onShow() → GET /v1/cart/list
  → 后端自动校验并返回 validItems + invalidItems
  → 前端分别渲染
```

若校验后有库存不足项，后端自动修正数量并移入失效区，前端接收后直接展示。
