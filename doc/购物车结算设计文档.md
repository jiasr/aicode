# 购物车结算设计文档

## 1. 概述

从购物车页点击「去结算」→ 确认订单页 → 提交订单 → 支付的全流程设计。

### 核心流程

```
购物车 → 选中商品 → 去结算
  → 订单确认页（选择地址/优惠券/留言）
    → 提交订单 → 创建订单（待支付）
      → 支付 → 支付成功 → 订单详情
```

## 2. 数据模型

### 2.1 订单主表 `t_mall_order`

```sql
CREATE TABLE t_mall_order (
    id                 INT PRIMARY KEY AUTO_INCREMENT,
    order_id           VARCHAR(64) NOT NULL UNIQUE       COMMENT '订单号（全局唯一）',
    user_id            VARCHAR(255) NOT NULL             COMMENT '用户ID',
    total_amount       INT NOT NULL DEFAULT 0            COMMENT '商品总金额(分)',
    discount_amount    INT DEFAULT 0                     COMMENT '优惠金额(分)',
    freight_amount     INT DEFAULT 0                     COMMENT '运费(分)',
    pay_amount         INT NOT NULL DEFAULT 0            COMMENT '实付金额(分)',
    pay_status         TINYINT DEFAULT 0                 COMMENT '支付状态 0未支付 1已支付 2已退款',
    order_status       TINYINT DEFAULT 0                 COMMENT '订单状态 0待付款 1已付款 2已发货 3已完成 4已取消',
    consignee_name     VARCHAR(100) NOT NULL             COMMENT '收货人姓名',
    consignee_mobile   VARCHAR(20) NOT NULL              COMMENT '收货人手机号',
    consignee_address  VARCHAR(500) NOT NULL             COMMENT '收货地址',
    remark             VARCHAR(500) DEFAULT ''           COMMENT '买家留言',
    payment_method     VARCHAR(32) DEFAULT ''            COMMENT '支付方式 wechat/balance',
    paid_at            DATETIME                          COMMENT '支付时间',
    create_time        DATETIME DEFAULT NOW(),
    update_time        DATETIME DEFAULT NOW() ON UPDATE NOW(),
    INDEX idx_user (user_id),
    INDEX idx_order (order_id),
    INDEX idx_pay_status (pay_status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单主表';
```

### 2.2 订单商品表 `t_mall_order_item`

```sql
CREATE TABLE t_mall_order_item (
    id              INT PRIMARY KEY AUTO_INCREMENT,
    order_id        VARCHAR(64) NOT NULL                COMMENT '订单号',
    spu_id          VARCHAR(255) NOT NULL               COMMENT '商品SPU ID',
    sku_id          VARCHAR(255) NOT NULL               COMMENT '商品SKU ID',
    title           VARCHAR(500) NOT NULL               COMMENT '商品标题',
    thumb           VARCHAR(500) DEFAULT ''             COMMENT '商品图片',
    spec_label      VARCHAR(200) DEFAULT ''             COMMENT '规格描述，如"5斤"',
    price           INT NOT NULL DEFAULT 0              COMMENT '成交价(分)',
    quantity        INT NOT NULL DEFAULT 1              COMMENT '购买数量',
    subtotal        INT NOT NULL DEFAULT 0              COMMENT '小计(分) = price * quantity',
    create_time     DATETIME DEFAULT NOW(),
    FOREIGN KEY (order_id) REFERENCES t_mall_order(order_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单商品明细表';
```

### 3.2 POST /v1/order/create

**请求体：**

```json
{
  "items": [
    { "spuId": "xxx", "skuId": "yyy", "quantity": 2 },
    { "spuId": "aaa", "skuId": "bbb", "quantity": 1 }
  ],
  "consignee": {
    "name": "张三",
    "mobile": "13800138000",
    "province": "广东省",
    "city": "深圳市",
    "district": "南山区",
    "detail": "科技园XX栋XX号"
  },
  "remark": "请放快递柜"
}
```

**逻辑：**

```
0. ⚠️ 所有价格以后端 DB 中 sku.price 为准，前端传的 price 不参与计算
1. 校验库存（所有 SKU），库存不足 → 返回错误
2. 生成订单号: CH{年月日}{8位流水号}（如 CH2026061400000001）
3. 查 DB 中每个 SKU 的 price，计算总金额 = Σ(DB_sku.price * quantity)
4. 创建 t_mall_order + t_mall_order_item
5. 删除购物车中已下单的 SKU
6. 返回订单号
```

**响应：**

```json
{
  "flag": true,
  "resData": {
    "orderId": "CH2026061400000001",
    "payAmount": 8970
  }
}
```

## 4. 订单号生成规则

```
CH + 年月日(8位) + 流水号(8位)
CH  20260614    00000001
```

- 使用数据库 `SELECT MAX(order_id) ...` 获取当天最大流水号 + 1
- 支持高并发（业务初期足够）

生成函数：

```python
def generate_order_id(user_id):
    prefix = datetime.now().strftime('CH%Y%m%d')
    # 从 DB 查当天最大订单号取后 8 位 + 1
    max_id = session.query(func.max(Order.order_id)).filter(
        Order.order_id.like(prefix + '%')
    ).scalar()
    seq = int(max_id[-8:]) + 1 if max_id else 1
    return '{}{:08d}'.format(prefix, seq)
```

## 5. 安全规则 — 价格后端计算

### ⚠️ 严禁前端传价格

前端请求体中**只传 skuId + quantity**，不传 price，后端根据 DB 中 `t_mall_goods_sku.price` 重新计算。

```json
// ✅ 正确 — 只传 skuId 和数量
{ "items": [{ "skuId": "yyy", "quantity": 2 }] }

// ❌ 错误 — 即使前端传了 price，后端一律忽略
{ "items": [{ "skuId": "yyy", "quantity": 2, "price": 1 }] }
```

### 后端校验清单

```python
for item in order_items:
    sku = session.query(GoodsSku).filter(
        GoodsSku.sku_id == item['skuId']
    ).with_for_update().first()

    # 价格取 DB，忽略前端
    real_price = sku.price   # ← 只用 DB 价格
    subtotal = real_price * item['quantity']
    total += subtotal
```

## 6. 库存扣减（悲观锁）

创建订单时扣减库存，使用 `SELECT ... FOR UPDATE` 防止超卖：

```python
with session.begin():
    sku = session.query(GoodsSku).filter(
        GoodsSku.sku_id == sku_id
    ).with_for_update().first()

    if sku.stock_quantity < quantity:
        raise Fail("STOCK_NOT_ENOUGH")

    sku.stock_quantity -= quantity
```

```

┌─ 收货地址 ───────────────────────┐

```

### 6.2 提交后

```
提交订单成功 → 跳转支付结果页
  → 支付成功：跳转订单详情
  → 未支付：留在支付结果页，可继续支付
```

- 点击「去结算」或单个商品结算时，未登录 → 弹窗提示登录
- 通过 `wx.login()` 静默登录后继续
